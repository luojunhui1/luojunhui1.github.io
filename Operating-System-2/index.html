Processes

The approach taken by modern operating systems is rely on a model which the execution of an application corresponds to the existence of one or more processes. We can think of a process as an entity that consist of a number of elements. Two basic elements of a process are program code and a set of data associated with that code. At any given point in time, while the program is executing, this process can be uniquely characterized by a number of elements, including the following:

  • Identifier
  • State
  • Priority
  • Program Counter
  • Memory Pointers: Include pointers to the program code and data
  • Context data
  • I/O Status Information
  • Accounting Information

The information in the preceding list is stored in a data structure named process control block, which has these characteristics:

  • It is possible to interrupt a running process and later resume execution as if the interruption had not
    occurred
  • Created and managed by the operating system
  • Key tool that allows support for multiple process

Process States

We can characterize the behavior of a processor by listing the sequence of instructions that execute for that process, such a listing is referred as a trace.

A Two-State Process Model

A process may be in one of two states:

  • running
  • not-running

When OS creates a new process, it creates a process control block for the process and enters that process into the system in the Not Running state. The OS knows the existence of the process, and waiting for a chance to execute. From time to time, the running process will be interrupted and the dispatcher will select a new process to execute, the former process will moves from the running-state to the not-running state, the latter process moves to the running state.

A Five-State Process Model

To descript process better, it is useful to split Not Running state into two states:

  • Ready: a process that is prepared to execute when given the opportunity
  • Blocked/Waiting: a process that cannot execute until some event occurs, such as the completion of I/O operation.

and there are two additional states that has been proven useful:

  • New: a process that has just been created and not been admitted to the pool of executable processes by the OS
  • Exit: a process that has been released from the pool of executable processes by the OS

The possible transitions among these five states are as follows:

  • Null:arrow_right:New:
  • New:arrow_right:Ready
  • Ready:arrow_right:Running
  • Running:arrow_right:Exit
  • Running:arrow_right:Ready: the most common reason for this transition is that the process has reached the maximum allowable time for uninterrupted of time discipline
  • Running:arrow_right:Blocked
  • Blocked:arrow_right:Ready
  • Ready:arrow_right:Exit: a parent may terminate a child process at any time

However, because the I/O is much slower than processor, there is still a possibility that all the processes in main memory may waiting for the completion of I/O operation at the same time even for multi-programed system, expanding the size of main memory cannot solve this problem, because the larger size of main memory will lead to larger size of processes but not more processes, the other solution is swapping, when none of the processes in main memory is in the Ready State, OS swap one of the blocked processes out on to disk into a suspend queue, this queue is out of main memory.

Normally, the OS would prefer to suspend a blocked process rather than a ready process, however, it may be necessary to suspend a Ready process if that there is a only way to free up a sufficiently large block of main memory, also, the OS may suspend a lower priority ready process rather than a higher priority blocked process.

Also, when there is a blocked/suspend process with a higher priority than any process in Ready state, and the OS believe that the blocking event for that process will occur soon, it would seem reasonable to bring a blocked process into main memory.

Process Description

If the OS is to manage processes and resources , it must have information about the current status of each process and resource, the OS constructs and maintains tables of information about each entity that is managing.

To manage and control a process the OS must know:

  • where the process is located
  • the attributes of the process that are necessary for its management

Typically, the collection of attributes is referred to as a process control block(PCB), which is the most important data structure in OS:

  • contains all of the information about a process that is needed by the OS
  • blocks are read and/or modified by every module in the OS
  • defines the state of the OS

Process Attributes:

  • Process identification
  • Processor state information
  • Process control information
