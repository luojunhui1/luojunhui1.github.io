互联网概述

现代社会有大家比较熟悉的三种网络，即电信网络、有线电视网络和计算机网络，这三者在最初设计时各有分工，但随着各自的发展，尤其是计算机网络的快速发展，电信网络和有线电视网络逐渐融入了计算机网络。从理论上来说，可以将三种网络融合为一种网络来提供所有的网络，这就是“三网融合”的概念。

“互联网”（Internet）和“互连网”（internet）概念并不一样，后者通常泛指由多个计算机网络互连而成的计算机网络,需要明确的概念是：网络把多个计算机连接在一起，而互连网把许多网络通过路由器连接在一起，所以互连网也是“网络的网络”，而前者通常特指当前全球最大的、开放的、由众多网络连接而成的特定的互连网，它采用TCP/IP协议族作为通信的规则，前身为美国的ARPANET。

互联网基础结构发展共有三个阶段，第一阶段是从单个网络ARPANET向互连网发展的过程，第二阶段是建成了三级结构的互联网，美国政府资助的NSFNET是一个三级计算机网络，分为主干网、地区网和校园网（企业网），在这阶段许多公司就已经接入互联网，美国政府决定将互联网的主干网交给私人公司来经营。第三阶段的特点是逐渐形成了多层次的ISP（Internet Service Provider）结构的互联网，根据提供服务的覆盖面积以及拥有的IP地址数目的不同，ISP也分为不同层次：主干ISP、地区ISP和本地ISP，随后，为了实现更快地转发分组以及更有效地利用互联网资源，互联网交换点出现了（IXP, Internet eXchange Point），IXP通常采用工作在数据链路层的网络交换机，这些网络交换机通常用局域网连接起来。

互联网从工作方式上来说可以分为边缘部分和核心部分，前者由用户直接使用的连接在互联网上的主机组成，后者由网络和连接这些网络的路由器组成，这部分为边缘提供连通性和交换。在互联网的边缘部分，端系统之间的通信方式可以分为两部分：客户-服务器(Client-Server )方式及对等方式（P2P, Peer-to-Peer），前者最重要的特征为客户为服务请求方，服务器是服务提供方；后者从本质上来说仍然是CS方式，只是对等连接中的每一台主机既是客户又是服务器。在互联网的核心部分，路由器是它的核心构建，它的作用是分组转发，采用存储转发技术，将报文分成一个个等长的数据段，加上一些必要信息组成的首部，就构成了一个分组（包），由于每个分组都包含了诸如目的地址和源地址等主要控制信息，每一个分组都能在互联网中独立地选择传输路径，并被正确地交付到分组传输的终点。分组传输时会一段一段地断续占用通信资源，这种方式非常适合传送突发式的计算机数据，省去了电路交换所需的建立连接和释放连接的开销，传输的效率就更高。但是因为分组交换无法保证通信时端到端的带宽，分组交换也带来了时延的问题，分组交换网也需要具有处理包头上的控制信息的能力。

互联网有两个最重要的特点，即连通性和资源共享，前者为用户提供了经济、便捷的交换信息的方式，后者让用户可以便捷地利用互联网上的各种资源。

计算机网络需要作为一个单独的概念被提出来，它指由一些通用的、可编程的硬件互连而成的网络，按不同的分类标准，它可以被分为：

  • 按网络作用的范围分类：广域网（WAN,Wide Area Network）、城域网(MAN,Metropolitan Area Network)、局域网(LAN,Local Area Network)、个人区域网(PAN,Personal Area Network)。
  • 按网络使用者分类：公用网、专用网

还有一种计算机网络比较特殊，它就是把用户接入到互联网的网络，即接入网，它本身既不属于互联网的核心部分也不属于边缘部分，从覆盖范围来看，很多接入网还属于局域网。

在计算机网络的概念中，分层次的体系结构是最基本的，这是为了将计算机网络面对的庞大而复杂的问题转化为若干较小的易于研究和处理的局部问题，分层可以带给我们好多好处，比如：

  • 各层之间是独立的
  • 灵活性好
  • 结构上可分隔开
  • 易于实现和维护
  • 能促进标准化工作

但是，分层也有一些缺点，例如有些功能会在不同层次中重复出现，因而产生了额外的开销。分层应使每一层的功能都尽量明确，层数太少就会使每层的协议太复杂，层数太多又会在描述和综合各层功能的系统功能任务时遇到较多的困难，通常各层要完成的任务主要包括以下这些：

  • 差错控制：使相应层次对等方的通信更加可靠。
  • 流量控制：发送端的发送速率必须使接收端来得及接受。
  • 分段和重装：发送端将要发送的数据块划分为更小的单位，在接收端将其还原。
  • 复用和分用：发送端高层绘画复用一条底层的连接，在接收端再进行分用。(目测只要某层具有上下层就要求实现该功能？)
  • 连接建立和释放：交换数据前先建立连接，数据传送结束后释放连接。

为了能使不同体系的计算机网络体系结构的用户 能够相互连接交互信息，ISO（国际标准化组织）提出了OSI/RM(Open Systems Interconnection Reference Model, 开放系统互连基本参考模型)，但在它得以完善之前，基于TCP/IP的互联网已经在全球范围内成功地运行了。网络协议试图明确网络中数据交换的数据的格式以及有关的同步问题，它主要由三个部分组成：

  • 语法：数据与控制信息的结构或格式
  • 语义：需要发出何种控制信息以及如何响应控制信息
  • 同步：对事件的实现顺序进行说明

OSI的七层体系结构理论完整但复杂又不实用，TCP/IP的四层体系结构得到广泛应用但实质上只能算是只有最上面的三层。仅在学习的时候，这里使用一种为介绍互联网结构而设计的5层协议的体系结构。

这里作为概述的一部分简述各部分的功能：

层 功能
应用层（application layer） 通过应用进程间的交互来完成特定的网络应用，定义应用进程间通信和交互的规则。
运输层(transport layer) 负责向两台主机中进程间的通信提供通用的数据传输服务，传输层最主要使用TCP(Transmission Control Protocol,数据传输控制协议)和UDP(User Datagram Protocol,用户数据报协议),
网络层（network layer） 负责为分组交换网上不同的主机提供通信服务，发送数据时，由网络层把运输层产生的报文段或用户数据封装成分组或包进行传送，在TCP/IP体系中，由于使用IP(Internet Protocol,网际协议)协议，所以在此二者条件符合的条件下，也将分组称作IP数据报。
数据链路层（data link layer） 在两个相邻结点之间传送数据时，数据链路层将网络层交下来的IP数据报组装成帧，在两个相邻结点间的链路上传送帧，每一帧包括控制信息和数据。在接受信息时，控制信息使接收端能知道一帧中开始和结束的位置及帧中有无差错。
物理层（physical layer） 在物理层上传送数据的单位使bit，物理层要考虑与传输媒介的接口有关的一些特性，比如机械特性、电气特性、功能特性和过程特性。注意，传递信息所使用的传输媒介并不属于物理层。

在了解了以上只是后，这里再对协议的内涵做更深入的解释：协议是控制两个对等实体进行通信的规则的集合。在协议的控制下，两个对等实体间的通信使得本层能够向上一层提供服务，要实现本层协议，还要使用下一层所提供的服务。从上面所说的“对等”的内涵可以看出，协议的作用实体是“水平”的，但“服务”是”垂直“的。

互联网体系结构-物理层

物理层的任务是实现在各种计算机传输媒体上传输比特流，而不是指具体的传输媒体，为了完成这一任务，可以将物理层的特性描述为以下4个方面：

  • 机械特性：指明接口所用的接线器的形状和尺寸、引脚数目和排列、排列和锁定装置等。

  • 电气特性：指明在接口电缆的各条线上出现的电压的范围。

  • 功能特性：指明某条线上的某一电平的电压的意义。

  • 过程特性：指明对于不同功能的可能的各种事件的出现顺序。

其实物理层的含义并不多，余下该说明的其实是物理层的协议，但为了理解上的方便，以下也对数据通信做简短的概述。

数据通信系统模型：

数据模型通信系统部分 组成部分 备注
源系统（发送端、发送方） 源点（原站、信源）；发送器 源点设备产生数据；发送器将源点生成的比特流编码以便于传输使用；
传输系统（传输网络） 简单的传输线或者复杂的传输网络
目的系统（接收端、接收方） 接收器；终点 接受传输系统传送过来的信号并将其解码还原；(书中说还原成发送端产生的比特流，但发送端输出的比特流实际上是经过编码的，应是源点产生的比特流才对？)从接收器获取传送过来的数字比特流并将其输出。

信道是向某一方向传输消息的媒体，与电路并不等同，一条电路中包含一条发送信道和接收信道。从信道双方信息交互的方式来看，可以有以下几种基本方式：

信道信息交互方式 含义
单向通信（单工通信） 只能有一个方向的通信而没有反方向的交互；只需一条信道
双向交替通信（半双工通信） 双方都可以发送消息，但不能同时；需两条信道
双向同时通信（全双工通信） 通信双方可以同时发送和接收消息；需两条信道

来自信源的信号称为基带信号，它通常包含很多低频成分甚至有直流成分，但许多信道并不能传输这种低频分量或直流分量，这就需要对基带信号进行调制，调制可分为两大类：基带调制和带通调制，前者对基带信号进行波形变换，使它能够与信道特性相适应，变换后的信号仍是基带信号，这个过程也称为编码；后者使用载波进行调制，将基带信号搬移到较高的频段，并转换为模拟信号，这样就能更好地在模拟信道中传输，经过载波调制后的信号称为带通信号。

对于基带调制来说，有以下四种常用的编码方式：

编码方式 含义
不归零制 正电平代表1，负电平代表0
归零制 正脉冲代表1，负脉冲代表0
曼彻斯特编码 位于周期中心的向上跳变代表1，向下跳变代表0
差分曼彻斯特编码 在每位的中心始终都有跳变，但在每一位的开始边界有跳变代表1，否则代表0

对于带通调制来说，有以下三种基本的调制方法：

带通调制方法 含义
调幅（AM,amplitude modulation） 载波的振幅随基带信号而变化
调频（FM,frequency modulation） 载波的频率随基带信号而变化
调相（PM,phase modulation） 载波的初始相位随基带信号而变化

从概念上说，限制码元在信道上的传输速率的因素主要有以下两个：

  • 信道所能通过的频率范围。书中没有对这几个问题进行解释：码间干扰现象和传输速率的关系；高频分量和信道频率范围的关系。结论是：若码元的传输速率超过了上限，就会出现严重的码间干扰问题，以至于接收端不可能识别码元。

  • 信噪比。信噪比指信号的平均功率与噪声的平均功率之比，记作S/N，常以dB做计量单位$dB=10\log_{10}{S/N}(dB)$,香农公式为：
    $$
    C = W log_{2}{1+S/N}\qquad(bits/s)
    $$
    其中，C是信道的极限传输速率，W为信道的带宽（HZ）。

对于物理层下的传输媒体，这里只会略过做简介：

传输媒体分类 示例
引导型传输媒体 双绞线；同轴电缆；光缆；
非引导型传输媒体 低频通信；短波通信（高频通信）；微波通信（特高频到极高频）：地面微波接力通信，卫星通信；红外通信；激光通信；

以下对信道复用技术做总结：

复用技术 含义
频分复用(FDM,frequency Division Multiplxing) 用户在相同时间占用不同的带宽资源
时分复用(TDM,Time Division Multiplexing) 用户在不同时间占用相同频带宽度
统计时分复用（异步时分复用） 使用STMD帧来传送服用数据，只有当一个帧放满了才发送出去
波分复用(WDM,Wavelength Division Multiplexing) 光的频分复用，此时分的是光波
码分复用（CMD, Code Division Multiplexing） 使用CMDA的每个站被指派一个唯一的码片序列,若该站要发送比特位1，则发送该码片序列，否则发送改码片序列的反码。为实现复用，每个站的码片序列必须正交，如此才能将每个站的信息分离，具体公式如下：

向量S表示站S的码片，若内积为1，则表示S站发送比特位1，若内积为-1，则表示S站发送比特位0，若内积为0，则表示S站未发送比特位。
