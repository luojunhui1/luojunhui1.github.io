Threads

The discussion so far has presented the concept of a process as embodying two characteristics:

  • Resource Ownership
  • Scheduling/Executing: process is a entity that can be scheduled and dispatched by OS

These two characteristics are independent and could be treated independently by OS, the unity of dispatching is usually referred to as a thread or lightweight process, while the unit of resource ownership is usually referred to as a process or task.

Multithreading

multithreading refers to the ability of an OS to support multiple, concurrent paths of execution within a single process.

In a multithreading environment, a process is defined as the unit of resource allocation and a unit of protection. The following are associated with processes:

  • a virtual address space to hold the process image
  • Protected access to processors, other processes, files and I/O resources

Within a process, there may be one or more threads, each with the following:

  • an execution state (Running, Ready, etc.)
  • saved thread context when not running
  • an execution stack
  • some per thread static storage for local variables
  • access to the memory and resources of its process (all threads of a process share this)

Benefits of Threads

Benefits of Threads
Less time to create a new thread than a process
Less time to terminate a thread than a process
Switching between two threads takes less time than switching between processes
Threads enhance efficiency in communication between programs by avoiding kernel intervention(In most OSes, the communication among threads requires the intervention of kernel)

Thread Execution States

The key states for a thread are:

  • Running
  • Ready
  • Blocked

Thread operations associated with a change in thread state are:

  • Spawn
  • Block
  • Unblock
  • Finish

Thread Execution

In a OS that supports threads, scheduling and dispatching is done on a thread basis, most of the states information dealing with execution, states are maintained in a thread-level data structure, suspending a process mean suspending all the threads in this process, so with the termination.

It is necessary to synchronize the activities of the various threads, because all the threads of a process share the same address space and other resources, any alteration of a resource by one thread may affect an other thread in the same process.

Thread Classification

Two Types of threads
User Level Thread(ULT)
Kernel Level Thread(KLT)

User Level Threads(ULTs)

All thread management is done by the application, the kernel is not aware of the existence of threads. Thread Library provides programmer with API for creating and managing threads.

Advantages of ULTs

  • Thread switching does not require kernel mode privileges
  • Scheduling can be application specific
  • ULTs can run on any OS

Disadvantages of ULTs

  • when a ULT executes a system call, not only is that thread blocked, but all of the threads within the process are blocked
  • In a pure ULT strategy, a multithreaded application cannot take advantage of multiprocessing

Kernel Level Threads (KLTs)

Thread management is done by the kernel, no thread management is done by the application.

Advantages of KLTs

  • The kernel can simultaneously schedule multiple threads from the same process on multiple processors
  • If one thread in a process is blocked, the kernel can schedule another thread of the same process
  • Kernel routines(内核例程) can be multithreaded

Disadvantage of KLTs

The transfer of control from one thread to another within the same process requires a mode switch to the kernel

Combined Approaches

Thread creation is done in the user space, Bulk of scheduling and synchronization of threads is by the application.

Multicore

The potential performance benefits of a multicore organization depend on the ability to effectively exploit the parallel resources available to the applications. Serial code may make a great influence on the performance of a multicore organization, and the communication and distribution of work to multiple processors and cache coherence also heavier the burden of processors, which results in a curve where performance peak and then begins to degrade.
